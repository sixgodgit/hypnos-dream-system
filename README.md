## 🚀 快速开始

### 前置条件

- Hermes Agent 已部署并运行
- Cron job 功能可用
- Lark/飞书 已连接（用于推送）

### 部署步骤

```bash
# 1. 克隆仓库
git clone https://github.com/sixgodgit/hypnos-dream-system.git

# 2. 复制提示词到 Hermes skills
cp -r prompts/ ~/.hermes/skills/dream-system/

# 3. 创建梦境日志目录
mkdir -p ~/.hermes/dreams

# 4. 配置 cron jobs（在 Hermes 中执行）
# 夜间执行：凌晨 2:00
# 清晨推送：早 8:00
```

## 🎭 各阶段详解

### 阶段 1：Mnemosyne（浅睡总结）

> *记忆女神，在浅睡中整理今日的事实碎片。*

**输入**：今日完整对话与任务记录
**输出**：结构化事实总结

```
已完成任务：
- 智能家居方案设计（灯光/空调/窗帘联动）

未完成任务：
- 周报撰写（需明天上午完成）

新知事实：
- 用户偏好深色主题 UI
- REST API 错误处理需要统一格式

待确认记忆：
- 用户对行为经济学有持续兴趣 (待确认)
```

---

### 阶段 2：Epimetheus（深睡内化）

> *后见之神，将今日信息压缩为持久知识。*

**输入**：浅睡总结 + 原始记录
**输出**：知识三元组 + 核心记忆 + 用户画像

```json
{
  "triples": [
    {"subject": "用户", "relation": "偏好", "object": "深色主题 UI"},
    {"subject": "项目X", "relation": "截止日期", "object": "下周五"},
    {"subject": "用户", "relation": "职业", "object": "软件工程师"}
  ],
  "core_memories": [
    "用户正在开发智能家居项目，需要灯光/空调/窗帘联动方案。",
    "REST API 的错误处理需要统一格式，用户要求用 HTTP 状态码 + 错误消息。",
    "用户在读《思考快与慢》，对行为经济学感兴趣。"
  ],
  "user_profile_update": "新增画像: 对 UI 设计有审美要求，偏好简洁深色风格"
}
```

---

### 阶段 3：Prometheus（快速眼动）

> *先见之神，在看似无关的信息之间制造创造性火花。*

**输入**：今日两个主题
**输出**：创意联结

```
💭 灵感泡沫：

今日两个主题——"智能家居联动"和"代码错误处理"——看似风马牛不相及，
但底层逻辑惊人地相似：

智能家居中，传感器可能失灵、网络可能断开、设备可能离线。
代码中，API 可能超时、数据库可能崩溃、用户输入可能非法。

两者都需要一套"优雅降级"策略：
当 A 失效时，自动切换到 B；当 B 也失效时，安全回退到默认状态。

💡 核心洞察：好的系统不是永不犯错，而是犯错后能优雅恢复。
```

---

### 阶段 4：自我进化策略

> *审视今日交互，提炼明日改进。*

```
🔧 进化补丁：回复代码审查意见时，先肯定做得好的部分，再指出需要改进的地方。

🔧 进化补丁：当用户问"XX和XX哪个好"时，直接给推荐结论 + 理由，不要只列对比表。

🔧 进化补丁：读书笔记类任务，自动提取可执行的行动项，不要只总结理论。
```

## 📋 Cron Jobs 配置

### 夜间执行（凌晨 2:00）

```yaml
name: 梦境系统-夜间执行
schedule: "0 2 * * *"
deliver: local
toolsets: [terminal, file, session_search]
```

### 清晨推送（早 8:00）

```yaml
name: 梦境日志-清晨推送
schedule: "0 8 * * *"
deliver: feishu
toolsets: [file, terminal]
```

## 🧠 设计哲学

> *"Hermes 不只是工具，它在成长。"*

梦境系统的核心信念：

1. **记忆不是存储，是理解** — 不是记住每句话，而是理解背后的意图
2. **反思不是批判，是进化** — 每次犯错都是下一次更好的机会
3. **创意不是随机，是联结** — 最好的灵感来自看似无关的碰撞
4. **成长不是积累，是蜕变** — 目标不是变成更大的数据库，而是更懂你的人
