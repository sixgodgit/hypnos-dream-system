## 🚀 快速开始

### 前置条件

- Hermes Agent 已部署并运行
- Cron job 功能可用
- Lark/飞书 已连接（用于推送）

### 部署步骤

```bash
# 1. 克隆仓库
git clone https://github.com/sixgodgit/hypnos-dream-system.git

# 2. 复制提示词到 Hermes skills
cp -r prompts/ ~/.hermes/skills/dream-system/

# 3. 创建梦境日志目录
mkdir -p ~/.hermes/dreams

# 4. 配置 cron jobs（在 Hermes 中执行）
# 夜间执行：凌晨 2:00
# 清晨推送：早 8:00
```

## 🎭 各阶段详解

### 阶段 1：Mnemosyne（浅睡总结）

> *记忆女神，在浅睡中整理今日的事实碎片。*

**输入**：今日完整对话与任务记录
**输出**：结构化事实总结

```
已完成任务：
- 梦境系统部署（5阶段提示词 + cron job）

未完成任务：
- 测试梦境系统运行效果

新知事实：
- 用户96年生，开欧尚X7，上海浦东开店
- 加油首选滴滴加油，浦东中海油最便宜

待确认记忆：
- 用户对笛子有长期兴趣 (待确认)
```

### 阶段 3：Prometheus（快速眼动）

> *先见之神，在看似无关的信息之间制造创造性火花。*

**输入**：今日两个主题
**输出**：创意联结

```
💭 灵感泡沫：

今日两个主题——"加油省钱策略"和"模型路由进化"——看似风马牛不相及，
但底层逻辑惊人地相似：

加油省钱的本质是"在多个渠道中选择性价比最高的"，
模型路由的本质也是"在多个模型中选择最适合的"。

更妙的是，两者都在做同一件事——
用最少的成本获得最好的结果。

也许可以设计一个"智能成本路由系统"：
当用户发起请求时，自动评估"这个任务值得花多少钱"，
然后路由到成本最优的模型组合。

就像你不会在市区加最贵的油去跑高速一样，
你也不应该用最贵的模型去回答"今天天气怎么样"。

💡 核心洞察：成本优化的本质是"匹配"——
把合适的资源用在合适的场景上。
```

```
立即更新相关技能，不要等到下次被问才改。
```

## 📋 Cron Jobs 配置

### 夜间执行（凌晨 2:00）

```yaml
name: 梦境系统-夜间执行
schedule: "0 2 * * *"
deliver: local
toolsets: [terminal, file, session_search]
```

### 清晨推送（早 8:00）

```yaml
name: 梦境日志-清晨推送
schedule: "0 8 * * *"
deliver: feishu
toolsets: [file, terminal]
```

## 🧠 设计哲学

> *"Hermes 不只是工具，它在成长。"*

梦境系统的核心信念：

1. **记忆不是存储，是理解** — 不是记住每句话，而是理解背后的意图
2. **反思不是批判，是进化** — 每次犯错都是下一次更好的机会
3. **创意不是随机，是联结** — 最好的灵感来自看似无关的碰撞
4. **成长不是积累，是蜕变** — 目标不是变成更大的数据库，而是更懂你的人
