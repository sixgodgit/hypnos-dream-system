# ⚡ 触发条件与调度策略

## 触发层级

### 第一层：即时触发

| 条件 | 动作 | 优先级 |
|:----:|:----:|:----:|
| 用户说"晚安" | 立即启动梦境系统 | 🔴 最高 |
| 用户说"睡了"/"去睡了" | 同上 | 🔴 最高 |
| 用户说"明天见" | 同上 | 🔴 最高 |

### 第二层：空闲触发

| 条件 | 动作 | 延迟 |
|:----:|:----:|:----:|
| 30分钟无交互 | 检查是否为深夜（22:00-06:00） | 30分钟 |
| 是深夜 | 启动梦境系统 | 立即 |
| 不是深夜 | 不触发 | — |

### 第三层：保底触发

| 条件 | 动作 | 时间 |
|:----:|:----:|:----:|
| 每日凌晨 2:00 | Cron 强制执行 | 02:00 |
| 当天无对话 | 输出"今夜无事，安眠" | — |
| 当天有对话 | 完整 5 阶段执行 | — |

## 调度策略

### 正常工作日

```
09:00 - 22:00  → 正常对话模式
22:00 - 02:00  → 可能触发即时/空闲触发
02:00          → 保底 Cron 执行
02:00 - 08:00  → 系统休眠
08:00          → 推送昨日梦境日志
```

### 周末/节假日

```
与工作日相同，但用户可能更晚入睡
空闲触发延迟可适当延长至 45 分钟
```

## 异常处理

| 异常 | 处理 |
|:----:|:----:|
| 当天无任何对话 | 输出"今夜无事，安眠"，不执行后续阶段 |
| session_search 失败 | 使用 memory 中的近期信息作为降级数据 |
| 某阶段执行失败 | 跳过该阶段，继续下一阶段 |
| 全部阶段失败 | 输出最小化总结，记录错误日志 |
| 日志文件写入失败 | 输出到 stdout，下次推送时补发 |
