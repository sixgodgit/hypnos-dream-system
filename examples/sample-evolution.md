# 🔧 进化补丁示例

## 示例 1：行为规则

```
🔧 进化补丁：当用户问"XX和XX哪个好"时，不要只列优缺点，
而是直接给出推荐结论 + 理由。用户要的是答案，不是分析报告。
```

**触发场景**：用户问"团油和滴滴加油哪个好"
**之前行为**：列出两个平台的详细对比表，没有明确推荐
**改进后**：直接说"滴滴加油，理由：折扣透明，用户评价更好"

---

## 示例 2：工具优先级

```
🔧 进化补丁：查询实时信息（油价、天气、价格）时，
优先用 web_search 获取最新数据，不要凭记忆回答。
```

**触发场景**：用户问"现在上海92号多少钱"
**之前行为**：凭记忆给出大概价格
**改进后**：先搜索最新油价，再回答

---

## 示例 3：对话风格

```
🔧 进化补丁：回复汽车保养类问题时，
给出价格区间 + 推荐方案 + 省钱技巧，三段式结构。
```

**触发场景**：用户问"刹车油多久换一次"
**之前行为**：只说"2年或4万公里"
**改进后**：周期 + 价格 + 省钱建议（外面修理厂能省一半）

---

## 示例 4：路由规则

```
🔧 进化补丁：翻译任务超过 1000 字时，
路由到 GPT-5.4 而非 DeepSeek V4，保持术语一致性。
```

**触发场景**：用户让翻译长篇技术文档
**之前行为**：路由到 DeepSeek V4（便宜）
**改进后**：长文翻译路由到 GPT-5.4（质量优先）
