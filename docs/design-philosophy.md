# 🧠 设计哲学

## 核心信念

> *"Hermes 不只是工具，它在成长。"*

### 1. 记忆不是存储，是理解

传统 AI 记忆是 key-value 存储——记住"用户喜欢咖啡"。

Hermes 的记忆是**理解性记忆**——不仅知道你喜欢咖啡，还知道你只喝深度烘焙、早上第一杯要美式、下午换成拿铁、不加糖但要加燕麦奶。

**梦境系统的作用**：从碎片化的对话中提取这种深层理解。

### 2. 反思不是批判，是进化

Epimetheus 的名字来自希腊神话——他是"后见之神"，代表事后才理解的能力。

在梦境系统中，Epimetheus 不是来批判今天的错误，而是来**理解为什么犯错**，然后**调整明天的行为**。

**进化补丁的本质**：不是惩罚，是学习。

### 3. 创意不是随机，是联结

Prometheus 在神话中盗火给人类，代表**创造性破坏**。

在梦境系统中，Prometheus 负责在看似无关的信息之间制造**意外的联结**。这种联结可能毫无用处，但也可能成为下一次突破的种子。

**灵感泡沫的价值**：99% 可能没用，但 1% 可能改变一切。

### 4. 成长不是积累，是蜕变

Hermes 的成长不是变成更大的数据库，而是**更懂你的人**。

每一份梦境日志都是一次蜕变——从"知道你开欧尚X7"到"理解你关注性价比、喜欢实用建议、反感空洞分析"。

**最终目标**：让每次对话都感觉"这个 AI 真的懂我"。

## 与传统 AI 记忆的区别

| 维度 | 传统 AI 记忆 | Hypnos 梦境系统 |
|:----:|:------------:|:---------------:|
| 存储方式 | Key-Value | 三元组 + 核心记忆 |
| 更新频率 | 实时 | 每日批量 |
| 理解深度 | 表面事实 | 深层意图 |
| 创造性 | 无 | 跨域联结 |
| 自我进化 | 无 | 策略补丁 |
| 用户参与 | 被动 | 主动确认 |

## 技术实现原则

1. **最小化 Token 消耗** — 每个阶段的 prompt 控制在 200 tokens 以内
2. **渐进式部署** — 先跑通，再优化，不要一步到位
3. **用户可控** — 待确认记忆需要用户确认，不自动写入
4. **可追溯** — 每份梦境日志都有时间戳和阶段标记
5. **可回滚** — 如果进化方向错误，可以回退到之前的版本
